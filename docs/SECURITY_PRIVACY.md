# 安全与隐私边界

## 数据分类

- 高敏感：token、AppSecret、JWT secret、密码、原始周报、通讯录、员工 userid。
- 受限：个人评价、缺交名单、团队负责人履职、部门范围和反馈内容。
- 内部：汇总统计、运行状态、无个人明细的系统指标。
- 公开：不包含公司或个人信息的代码与通用文档。

## 规则

- `.env`、`config/.env`、`output/` 和 `logs/` 不进入 Git。
- 默认测试只使用虚构姓名、虚构 userid 和临时目录。
- 真实免交名单只保存在未跟踪的 `config/.env`；源码、示例、测试、普通日志和任务记录不得写入真实姓名或 userid。
- 真实负责人覆盖关系只保存在服务器未跟踪的 `.env`；运行时以精确唯一 selector 解析为 userid，歧义或缺失必须失败关闭，且不得进入 Codex 子进程环境。
- 异常信息不得包含 access token、完整 API 响应或原始周报正文。
- 数据下载、Markdown 报告和统计 API 使用相同的服务层权限范围。
- 单人周报原文接口必须先从服务层过滤后的提交状态定位稳定 `userid`，范围外统一按不存在处理；每次只返回一人的文本字段，不返回原始 JSON、附件、内部路径或批量正文。响应前清洗常见 token/secret/password 形态并限制文本总量，超限只返回安全状态。
- 负责人附件名、附件正文和履职评价属于受限数据；原始负责人输入与正式履职表必须逐行应用同一服务层范围过滤，日志不得输出 fileId、spaceId 或下载地址。
- 只下载负责人附件，并限制为 HTTPS、受支持扩展名、单文件 20 MB 和每周 100 MB；文件仅写入忽略的 `output/<week>/attachments/team_leads/`，目录使用 userid 哈希，分析包只记录本地相对路径。
- 管理员账号管理能力与完整周报读取能力分离。
- 生产模式默认启用，必须显式配置 `WEEKLY_JWT_SECRET` 和 `WEEKLY_BOOTSTRAP_ADMIN_PASSWORD`；缺失、空值或开发默认值会阻止启动，错误只显示变量名。
- 本地开发只有显式设置 `WEEKLY_AUTH_DEVELOPMENT_MODE=true` 时才能使用开发默认凭据；Docker 部署不得启用该标志。
- 前端 API 错误只采用受限的 JSON `error` 字段，并清洗当前 token、请求密码及常见 Bearer/JWT 形态；不得拼接完整响应。
- Excel 导出只能使用后端已按当前账号范围过滤的数据；提交状态 XLSX 由 Spring 服务层使用标准 OOXML 库生成，不接受前端提交周报正文，且排除 userid、report_id、token、路径和原始周报正文。单元格统一写为字符串，避免把员工文本当作公式执行。
- 原周报 XLSX 属于高敏感下载：默认从服务器私有 `raw/all_reports.json` 读取，W28 经用户确认可在该文件缺失时回退 `raw/reports.json`；两条路径都必须先执行当前账号服务层范围过滤，再保留授权人员的 userid、原始字段和重复提交。文件不包含 report_id、内部路径、原始 JSON、地址、评论、点赞或阅读统计；除 W28 双模板完整性例外外，缺少完整快照、身份字段或内容超过 XLSX 单元格限制时拒绝生成。前端请求不携带正文或人员清单，普通日志只记录通用失败，不记录文件内容。
- 项目明细 CSV 的 userid、部门只用于服务层复用现有权限过滤；姓名仅在过滤完成后随授权项目记录进入公开八列 JSON 与 XLSX。工时和费用仅在严格匹配普通数字时写入数值单元格，其他内容按字符串写入，禁止公式执行。
- 自动通知试发接口仅允许 ADMIN；请求中的确认姓名必须与服务器反馈接收人精确一致且只解析到一个 userid。接口不接受 userid，不返回接收标识、凭据或钉钉原始响应，也不触碰正式周日/周一状态文件。
- 自动评价 Codex 只接收当前业务周的分析包、提交 CSV、负责人附件和政策快照；子进程移除 DingTalk/MySQL/JWT/免交及通用 secret/token/password 环境变量，仅允许临时 `workspace-write`、无审批且禁用网络。模型输出必须通过章节、人员覆盖和敏感字段校验后才写入正式报告。
- 自动评价的私有员工反馈只允许覆盖已提交稳定 userid；反馈正文不得含姓名、其他 userid、同事信息、原始周报段落、URL、secret、路径或附件标识，并受字段长度和总文件大小限制。该文件只保存在忽略的目标周自动化目录，不以原始结构进入 Web API 或 Git；仅 ADMIN 可通过受控复核服务取得与正式发送模板一致的单周通知正文。
- 自动评价如需代理，只允许通过私有 `WEEKLY_CODEX_BASE_URL` 显式传入 HTTPS 地址；Harness 仅显式配置 `crs`/`responses` provider 并读取私有 `CRS_OAI_KEY` 环境变量，不直接加载完整 Codex 用户配置，避免本地规则、路径或其他凭据进入自动任务。
- 反馈通知失败时返回最小必要信息，不泄露内部凭据或通讯录。
- 周日提醒只向每位缺交人员发送不含他人信息的私人工作通知；管理员结果只包含周次、汇总人数和安全错误原因，不包含姓名、userid、周报正文或原始钉钉响应。
- 周一评价只向每位已提交员工单独发送其本人经校验的精简反馈；暖心感谢与表扬、改进使用同一隐私规则，不得包含姓名、其他人员、客户/项目标识、原文长段、链接、路径或凭据。HR 联系人来自服务器私有配置。通知状态只记录阶段、人数、时间和可选反馈摘要，发送不确定时不得自动重发；管理员成功/失败推送结果不得包含人员或评价正文。
- Feedback 复核接口仅允许 ADMIN，且不等于完整周报权限。接口复用正式候选校验和消息格式化，每次最多返回一个周次；只在状态 `COMPLETE`、已发人数等于应发人数且摘要或旧状态时间一致时返回姓名、部门、职位和已发送模板正文。响应排除 userid、report_id、路径、原始周报、原始私有 JSON 和独立 HR 配置；普通日志不记录通知正文或完整响应。
- 提醒候选 userid 只存在于受控进程输出和内存中，不写普通日志；`output/<week>/reminders/sunday-1800.json` 只保存阶段、汇总人数和时间。远程发送结果不确定时不得自动重发员工通知。

## 变更审查

涉及认证、授权、下载、钉钉 API、反馈通知或员工内容的任务，交付时必须说明：数据读取范围、写入位置、日志内容、拒绝行为和测试证据。
