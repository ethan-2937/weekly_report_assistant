# Task: 下载与钉钉原周报明细结构一致的 XLSX

## 问题

现有下载只提供提交状态和项目明细，不能按钉钉原始导出结构保留新旧周报模板、重复提交和完整正文。参考文件包含两个工作表；当前采集只保存指定模板，直接从提交状态导出会遗漏旧模板和重复提交。

## 范围

- 包含：采集新旧周报模板的私有原始快照；按服务层权限生成原周报 XLSX；复刻参考表头、列宽、字体、边框、换行和行高；前端下载入口。
- 不包含：地址、评论、点赞、已读/未读及已读率；图片二进制下载；改变现有提交统计和 AI 评价口径。

## 验收标准

- [ ] 下载包含“优智科技周报（Weekly Outcomes ）”和“周报”两个工作表，并保留同一人员的多次提交。
- [ ] XLSX 只包含当前账号有权查看的报告；无全量快照时拒绝生成，不静默返回缺失数据。
- [ ] 不接受前端正文，不输出 report_id、内部路径、token 或原始 JSON；公式形态员工文本作为普通字符串。
- [ ] 导出行数与授权原始报告记录数一致，关键字段超限或身份缺失时失败关闭。

## 约束

- 相关产品不变量：周次使用既有周四至下一周四前的提交归属窗口；统计仍只使用指定模板。
- 相关架构边界：Python 采集原始快照，Spring 服务层过滤并生成 XLSX，Vue 只触发下载。
- 需要保留的无关工作区修改：无。

## 验证

- 测试：Python 65 项、Spring 88 项、Vue 47 项全部通过；覆盖双模板采集隔离、格式、重复行、权限、缺失快照、安全单元格和下载行为。
- 手工/集成证据：使用虚构数据生成并渲染两个工作表，确认 22/11 列结构、换行、列宽和删除字段；前端生产构建通过。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：删除参考文件中的地址、评论、点赞和阅读统计七列；保留历史为空的新模板字段。
- 剩余风险：历史周次需要重新采集后才有双模板全量快照；钉钉未返回的图片和工号保持空白。
- 延后工作：如钉钉后续开放互动统计接口，再单独评估恢复相关列。
