# Task: 本周重点恢复人员条目

## 问题

W29 正式评价采用逐人标题和五维列表时，前端只从员工评价表格提取 AI 红黑榜，导致本周重点只剩团队级拍板事项，人员条目被漏掉。

## 范围

- 包含：兼容表格与逐人叙述评价、使用授权提交名单补充部门/职位、前端回归测试。
- 不包含：重新生成 W29 评价、读取真实周报、改变后端权限或评价结论。

## 验收标准

- [x] 表格格式仍展示人员 AI 红黑榜及具体内容。
- [x] `### 姓名` 加五维列表格式也展示人员 AI 红黑榜。
- [x] 人员条目补充当前授权范围内的部门和职位，重名时不猜测资料。
- [x] 团队级拍板事项与人员级红黑榜同时保留。

## 约束

- 相关产品不变量：只使用当前账号已授权的提交人员，不扩大数据范围。
- 相关架构边界：前端只兼容展示格式，不修改报告正文或后端数据。
- 需要保留的无关工作区修改：无。

## 验证

- 测试：Python 70 项、Spring 89 项、Vue 49 项全部通过；覆盖 MarkdownReport 表格、叙述格式和重名行为。
- 手工/集成证据：Harness 检查和前端生产构建通过；仅使用虚构姓名、部门、职位和评价内容。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：前端从两种合法输出格式提取人员证据，现有 W29 无需重评。
- 剩余风险：如果正式报告完全没有逐人 AI 红黑榜结论，前端不能自行补造。
- 延后工作：无。
