# Task: 根据钉钉部门生成负责人下属确认名单

## 问题

当前评价 Skill 将个人工作评价写成五个管理维度；负责人履职输入只记录负责人身份、部门和附件证据，没有确定性的负责人-下属映射。用户确认：没有下属 userid 列表时，按钉钉通讯录所属部门生成映射，并在负责人下属未提交时判定负责人履职不合格；开发后提供各负责人下属名单供上级确认。

## 范围

- 包含：保留五维规则来源说明；按稳定 userid 优先、部门归属回退生成负责人下属映射；输出下属名单、映射来源和未交人数；将未交下属作为负责人履职的确定性风险输入；补充虚构数据测试和 Skill/文档说明。
- 不包含：读取真实 output/logs；猜测跨部门汇报关系；修改钉钉模板；在用户确认前把部门映射写成组织正式事实；本次不改前端视觉切片。

## 验收标准

- [x] 每个确定性负责人都有下属映射来源、下属名单和待确认标记。
- [x] 有显式下属 userid 列表时优先使用；没有时按共享钉钉部门归属回退，并排除负责人本人和免交人员。
- [x] 映射下属存在未交周报时，负责人履职输入明确标记不合格风险；映射为空或存在歧义时不得臆测。
- [x] 测试使用虚构姓名、userid 和部门，不读取真实周报或凭据。
- [x] Skill、输出格式、任务记录和质量检查同步更新。

## 约束

- 相关产品不变量：提交归属仍按周四至下一周周三；免交人员不进入统计；userid 优先于姓名。
- 相关架构边界：Python 生成确定性映射，Skill 只解释证据，Spring/Vue 不自行推断组织关系。
- 需要保留的无关工作区修改：不修改 `weekly_report_template.txt`，不扩展历史 `web/backend/`。

## 验证

- 测试：显式 userid 映射、部门回退、多部门、负责人本人排除、免交人员排除、未交下属风险和歧义状态。
- 手工/集成证据：检查生成的负责人下属确认表只位于忽略的 output 目录；不在仓库读取真实名单。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：五维标准继续作为当前管理评价规则；四维改造另立契约任务。部门回退只作为“待上级确认”的确定性候选，不冒充正式汇报关系。
- 剩余风险：钉钉通讯录可能没有完整汇报线或负责人管理多个部门，需要上级确认后再决定是否固化名单。
- 延后工作：用户确认后的正式负责人范围配置、前端确认名单页面和跨部门矩阵关系。
