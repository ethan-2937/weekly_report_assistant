# Task: 管理报告禁止内部标识标签

## 问题

W29 强制重评时，模型在管理报告的数据质量说明中写出了内部标识字段名，触发 `INTERNAL_IDENTIFIER_LABEL_EXPOSED`。Harness 正确拒绝了结果，但原提示词只要求不暴露值，没有明确禁止标签文字本身。

## 范围

- 包含：强化 Harness 提示词、Skill 和输出格式说明，补标签级隐私回归测试。
- 不包含：放宽隐私校验、修改真实周报、读取被拒绝的模型输出或改变通知逻辑。

## 验收标准

- [x] 管理报告不得出现四类内部标识字段名的任何大小写写法。
- [x] 身份匹配说明使用“内部稳定标识”，附件说明使用“附件标识”。
- [x] 私有反馈结构所需的身份键保持不变。
- [x] 被拒绝结果不覆盖上一份有效报告。

## 约束

- 相关产品不变量：正式报告与私人反馈继续通过既有隐私校验后原子发布。
- 相关架构边界：模型遵循输出契约，Harness 保持机械拒绝而不自动擦除可疑内容。
- 需要保留的无关工作区修改：无。

## 验证

- 测试：Python 71 项、Spring 90 项、Vue 49 项全部通过；渲染提示词含明确禁令，仅出现字段标签、没有值时仍被拒绝。
- 手工/集成证据：Harness 检查和前端生产构建通过；不读取服务器输出或真实周报。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：强化生成约束，不降低现有隐私门槛。
- 剩余风险：模型仍可能偶发违约；Harness 会继续保留旧报告并返回安全错误码。
- 延后工作：无。
