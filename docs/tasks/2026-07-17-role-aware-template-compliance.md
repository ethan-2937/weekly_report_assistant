# Task: 按岗位动态计算周报模板填写正确率

## 问题

提交概览当前固定按四个全员通用字段计算模板填写正确率。新模板已经明确技术、产品、销售岗位还有条件必填字段，导致这些岗位漏填项目、工时或费用时仍可能显示 100%。

## 范围

- 包含：基于通讯录职务的岗位识别、4/9/11 项动态分母、缺失项和前端规则说明、Skill 与回归测试。
- 不包含：改变 AI 四个前端主维度、增加 AI 评分项、推断多项目工时或费用分摊。

## 验收标准

- [x] 其他岗位按四个全员通用字段计算。
- [x] 销售岗位按通用四项及产品线、客户、项目、差旅、招待共九项计算。
- [x] 技术/产品岗位再加入工时合计和工时明细，共十一项计算。
- [x] `0` 费用视为已填写，空文本、空数组、空对象和 null 仍视为缺失。
- [x] 概览显示动态百分比、具体缺失项和岗位适用规则，不改变权限或评价维度。

## 约束

- 相关产品不变量：模板合规与管理评价分离；仅使用已授权提交行中的职务和原始字段。
- 相关架构边界：Spring 负责确定性合规计算；Vue 只呈现结果；Skill 描述相同口径。
- 需要保留的无关工作区修改：无。

## 验证

- 测试：Spring 岗位分母与缺失字段、Vue 规则和说明、现有权限与回归测试。
- 手工/集成证据：提交概览文案与百分比显示检查。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`
- 结果：2026-07-17 完整验证通过；Python 63 项、Spring 73 项、Vue 40 项，前端生产构建成功。

## 交付

- 决策：只对职务命中明确技术、产品、销售关键词的人员增加条件必填项；其他岗位不猜测。
- 剩余风险：新出现且不含已知关键词的职务需补充岗位识别规则。
- 延后工作：不建立真实人员级岗位覆盖名单，不提交通讯录或人员隐私。
