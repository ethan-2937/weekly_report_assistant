# Task: 管理员复核每周 Feedback 通知

## 问题

服务器上的通知目录可能由容器用户持有，人工生成预览文件会因权限失败；管理员需要在 Web 中按周核对每位员工实际收到的周一评价通知。

## 范围

- 包含：ADMIN 专属查询接口、发送正文共享格式化、历史一致性判断、按周加载和逐人折叠界面。
- 不包含：重发通知、修改评价正文、读取原始周报、向普通周报账号开放。

## 验收标准

- [x] 仅 ADMIN 可以查询已完整发送周次的通知正文。
- [x] 响应不包含 userid、文件路径、token、原始周报或私有配置字段。
- [x] 页面按需加载，逐人展示姓名、部门、职位和与发送端一致的完整正文。
- [x] 无法确认历史一致性时不把当前内容标记为已发送正文。
- [x] 非法周次、未完成发送、人数不一致和反馈校验失败均安全拒绝。

## 约束

- 相关产品不变量：ADMIN 通知审计能力不等于完整周报查看权限。
- 相关架构边界：Controller 依赖 Service；候选反馈继续由既有 Provider 校验；前端复用统一 API client。
- 需要保留的无关工作区修改：全部保留，不回退或清理。

## 验证

- 测试：Python 71、Spring 100、Vue 60 全部通过；新增服务、状态兼容、权限、API 和组件测试。
- 手工/集成证据：管理员面板按需加载并逐人折叠；普通角色不渲染面板，接口权限测试返回 403。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1` 已通过。

## 交付

- 决策：旧状态用文件时间做兼容判断；新状态保存反馈摘要，正文仍不落入通知状态文件；发送与复核共用同一格式化器。
- 剩余风险：旧周次没有摘要，只能在私有反馈文件未晚于完成时间时确认。
- 延后工作：如需长期不可变审计，可另建加密、受控保留的逐条发送快照。
