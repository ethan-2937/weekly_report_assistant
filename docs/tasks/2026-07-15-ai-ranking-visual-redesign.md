# Task: AI 红黑榜信息层级清晰且视觉均衡

## 问题

当前 AI 红榜和黑榜沿用普通 Markdown 标题与列表样式，人员身份、评价结论和具体证据挤在同一段文字中，红黑区域的视觉边界也不够明确。

## 范围

- 包含：本周重点中的 AI 红榜/黑榜结构化呈现、桌面双栏与移动端单栏、相关前端回归测试。
- 不包含：评价生成规则、后端接口、权限、登录、周次和报告原始内容。

## 验收标准

- [x] 红榜使用红色标题和浅粉色区域，黑榜使用黑色标题和浅灰色区域，正文均为黑色。
- [x] 姓名、部门/职位、评价结论和具体内容具有清楚层级，缺少部分字段时仍正常展示。
- [x] 红榜在上、黑榜在下，以全宽横向卡片展示，窄屏自动改为单列内容。
- [x] 两个榜单默认展开并可独立收起、展开，按钮状态可被辅助技术识别。
- [x] 不改变现有内容排序、权限和敏感信息清洗行为。

## 约束

- 相关产品不变量：前端只呈现后端已授权的正式报告内容。
- 相关架构边界：不新增 API，不在前端进行权限裁决。
- 需要保留的无关工作区修改：`docs/MULTI_AGENT_COORDINATOR_PROMPT.md`。

## 验证

- 测试：MarkdownReport 组件结构与内容回归、完整 Vitest 28 项通过。
- 手工/集成证据：1280px 下红榜在上、黑榜在下且人员/内容横向分栏；375px 自动单列，无横向溢出或控制台错误；独立折叠状态验证通过。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：将相邻的 AI 榜单标题和列表组合成全宽横向展示块，红榜在上、黑榜在下并可独立折叠，不改变原始 Markdown 和评价数据。
- 剩余风险：超长单条证据会增加卡片高度，但不会截断内容。
- 延后工作：无。
