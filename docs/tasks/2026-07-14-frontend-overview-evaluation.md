# Task: 前端提交概览与 AI 评价完成分区改造

## 问题

当前提交概览与 AI 评价仍集中在 `App.vue`，负责人文案不统一，正式五维评价缺少清晰的四维主展示边界，组件行为回归测试不足。

## 范围

- 包含：提交概览、未交跳转、周次日期、规则提示、负责人展示、AI 评价折叠与四维主展示、第五维详情保留、Vitest 回归、应用接线和局部样式。
- 不包含：后端/Skill 五维评价规则、权限模型、钉钉 OAuth 协议、真实运行数据、历史 Java 后端和受保护模板。

## 验收标准

- [x] 前端主视图仅展示四个评价维度，第五维保留在摘要或详情。
- [x] 所有“负责人候选”用户文案统一为“负责人”，底层兼容字段不变。
- [x] 未提交统计可跳转到未交名单；周次可见日期范围与规则说明完整。
- [x] 员工评价和负责人履职默认折叠，本周重点包含 AI 红榜与黑榜。
- [x] 登录、钉钉 OAuth、角色菜单、前端权限守卫和统一 401 行为保持。
- [x] 旧版或缺字段 Markdown 安全降级，不输出预处理内容。

## 约束

- 相关产品不变量：完整 ISO 周；周四至下周四前的提交归属窗口；服务层权限；模板合规与管理评价分离。
- 相关架构边界：Vue 只呈现；认证/API 继续集中在 `src/api/` 与 `src/composables/useAuth.js`。
- 需要保留的无关工作区修改：未跟踪的 `docs/MULTI_AGENT_PROMPTS.md`。

- 结果：`npm test -- --reporter=verbose` 通过，共 7 个测试文件、20 项测试；`npm run build` 通过。
- 结果：`git diff --check` 通过；可见文案扫描仅保留底层字段、旧内容清洗和否定断言中的兼容字样。
- 结果：展示组件按职责拆出解析、结构和局部样式模块后，`scripts/check-harness.ps1` 与 `scripts/verify.ps1` 均通过；未提高质量预算。
- 规模：`App.vue` 由基线 1324 行降至约 1129 行；`styles.css` 由 2288 行增至 2309 行；本任务未修改 baseline。

## 交付

- 决策：后端/Skill 五维保持不变；前端通过组件映射展示四个主维度，并在详情保留综合结论/需跟进。
- 决策：周报接口由 `useAuth` 的统一 `request` 注入周次 API wrapper，再统一做字段归一化；未新增 token、401 或权限分支。
- 决策：提交概览独占周次选择器，未交与 AI 评价页保留紧凑周次条；未交卡沿用现有状态页权限守卫与过滤字段。
- 剩余风险：正式评价内容只在 `isManagerReport === true` 时展示；旧 Markdown 没有标准二级标题时只能安全降级，无法自动折叠。
- 剩余风险：生产构建仍有既有单包超过 500 kB 警告，不在本次最小接线范围内处理。
- 延后工作：结合实际加载性能评估路由懒加载与分包，处理既有大 chunk 技术债。
- 提交与推送：本次由主协调 Agent 完成最终审查；集成 Agent 未提交、未推送。
