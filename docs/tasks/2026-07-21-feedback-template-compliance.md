# Task: 周一 Feedback 展示模板符合度并评估钉钉单聊

## 问题

周一个人 Feedback 当前只包含表扬、改进和感谢，没有展示提交概览使用的模板符合百分比。现有逐人发送虽然每次只包含一个 `userid`，但底层使用企业工作通知，仍可能被考勤等通知刷过。

## 范围

- 包含：复用 `TemplateComplianceService` 为已提交员工补充同源百分比；在正式通知和管理员复核共用模板中展示百分比；缺失或越界时失败关闭；增加默认关闭的企业机器人单聊通道。
- 不包含：替用户开通机器人权限、创建真实单聊会话或修改服务器私有凭据；访问真实周报、通讯录或钉钉凭据。

## 验收标准

- [x] 周一正式 Feedback 和管理员复核正文均显示与提交概览同源的 `0%` 至 `100%` 模板符合度。
- [x] 缺少或越界百分比时不发送员工通知，不把未知值显示为 `0%`。
- [x] 原有稳定 userid、人员覆盖、隐私校验、幂等和管理员安全结果行为不变。
- [x] 默认仍使用工作通知；显式启用机器人单聊后按人发送，失败时不回退工作通知。

## 约束

- 相关产品不变量：模板合规与管理评价分离；通知只发给已提交且私有反馈通过校验的稳定 userid。
- 相关架构边界：百分比由 Spring 的共享模板合规服务确定，Codex 和前端不重复计算。
- 需要保留的无关工作区修改：`docs/PROJECT_DEVELOPMENT_HISTORY.md`、`docs/tasks/2026-07-20-development-history-through-w29.md`。

## 验证

- 测试：相关定向 24 项通过；统一验证通过，其中 Python 71 项、Spring 106 项、Vitest 60 项，并完成 Vite 生产构建。
- 手工/集成证据：钉钉官方 `BatchSendOTO` 要求企业内机器人发送消息权限、机器人 `robotCode` 和有效机器人单聊会话；每次最多 20 个 userid。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：交付同源百分比和显式 `ROBOT_OTO` 通道；默认保持 `WORK_NOTICE`。只有所有员工建立有效机器人会话并完成单收件人试发后才在服务器启用，失败时不得自动回退造成重复发送。
- 剩余风险：格式模板改变后，旧的已发送状态摘要无法与新模板一致，管理员复核会安全显示内容变化警告。
- 延后工作：机器人应用开通、权限审批、全员建立会话和真实环境试发。
