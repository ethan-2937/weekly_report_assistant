# Task: 新周报模板生成独立项目明细模块并支持下载

## 问题

钉钉“优智科技周报（Weekly Outcomes）”新增产品线、客户、项目、工时和费用字段，现有采集输出、模板镜像和 Web 界面没有对应的结构化项目明细。

## 范围

- 包含：模板镜像与 Skill、确定性项目明细 CSV、受权限过滤的 Spring 接口与 XLSX、独立前端模块和回归测试。
- 不包含：读取历史真实周报、推断多项目费用分摊、修改登录/OAuth/角色权限模型。

## 验收标准

- [x] 新模板字段按截图顺序进入受保护模板镜像，通用四项评价口径保持不变。
- [x] 每份适用周报生成一条项目明细，多值字段保留原填写，不猜测拆分或费用分摊。
- [x] 项目明细列表和 XLSX 均应用现有服务层数据范围，公开结构保持参考工作簿 7 列。
- [x] 前端新增独立“项目明细”模块，支持周次、搜索、汇总和下载，旧周次显示空状态。
- [x] 失败响应不泄露正文、userid、路径或凭据。

## 约束

- 相关产品不变量：模板合规与管理评价分离；后端服务层决定可见范围。
- 相关架构边界：Python 生成确定性输出；Spring 过滤和导出；Vue 仅呈现。
- 需要保留的无关工作区修改：`docs/MULTI_AGENT_COORDINATOR_PROMPT.md`。

## 验证

- 测试：Python 解析、Spring 权限/导出、Vue API/组件/菜单。
- 手工/集成证据：参考工作簿渲染检查、前端宽屏与移动端检查。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`
- 结果：2026-07-17 全量验证通过；Python 63 项、Spring 69 项、Vue 40 项，前端生产构建成功。
- 视觉结果：XLSX 渲染无公式错误；项目明细页在 1280px 与 390px 下无页面级横向溢出。

## 交付

- 决策：服务端 CSV 中姓名、userid、部门只用于权限过滤，接口与 XLSX 不返回这些隐藏字段。
- 剩余风险：字段名称若在钉钉再次调整，需要同步别名或通过新测试报告确认。
- 延后工作：不对多个项目之间的工时和费用做自动分摊。
