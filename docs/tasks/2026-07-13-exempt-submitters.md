# Task: Configurable weekly-report exemptions

## 问题

当前 `scripts/run_weekly.py` 将钉钉授权通讯录中的全部成员作为应交候选，没有免交名单入口。用户明确要求部分人员不进入周报统计；若直接把真实姓名写入源码，会把受限员工信息提交到 Git。

## 当前实现证据与风险分析

- `write_submission_outputs` 直接遍历全部通讯录成员，生成 CSV、缺交 Markdown 和分析输入。
- 后端和前端读取这些确定性产物，因此只在页面隐藏人员会造成统计与下载结果不一致。
- 仅按姓名匹配可能受重名影响，应优先支持稳定 `userid`；姓名只作为精确兼容规则。
- 免交人员若提交周报，也不能重新进入应交人数、已交人数或分析输入统计。
- 真实免交名单属于受限员工信息，不进入 Git、测试快照或普通日志。

## 范围

- 包含：新增服务器私有免交配置解析；在确定性输出前同时过滤通讯录人员和对应报告；补充纯单元测试；同步示例配置、产品/架构/质量/隐私和 Skill 说明。
- 不包含：读取或修改真实通讯录、真实周报和 `output/`；改变钉钉模板；修改后端数据权限；自动推断免交人员；回填历史周报产物。

## 验收标准

- [x] `USERID:` 规则按稳定 userid 精确排除。
- [x] `NAME:` 和兼容普通文本规则按规范化后的完整姓名精确排除，不做模糊匹配。
- [x] 多条规则合并生效，未命中人员仍保留在应交统计中。
- [x] 免交人员的提交不进入提交人数、缺交名单或分析输入。
- [x] 空配置保持现有行为；非法前缀以不含人员标识的错误拒绝运行。
- [x] 测试只使用虚构姓名、userid、部门和报告内容。

## 约束

- 相关产品不变量：应交名单必须来自授权通讯录或明确名单；稳定 userid 优先；缺交判断不得猜测。
- 相关架构边界：Python 在生成 CSV/Markdown 前完成应交候选过滤；Spring/Vue 继续消费相同产物。
- 需要保留的无关工作区修改：开始实现时工作区干净；后续若出现未知改动立即停止。

## 验证计划

- 测试：`python -m unittest discover -s tests -p "test_*.py"`
- 静态检查：`python -m compileall -q scripts codex-skills/weekly-report-assistant/scripts`
- 隐私检查：确认 diff 不含用户提供的真实免交名单、凭据、原始周报或通讯录。
- 统一命令：`powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## 交付

- 决策：真实免交名单保留在未跟踪的 `config/.env`，仓库只版本化通用规则和虚构示例。
- 验证：13 个 Python 测试、27 个 Spring 测试、8 个 Vitest 测试、前端生产构建、Harness 检查和 `scripts/verify.ps1` 全部通过。
- 复杂度：`run_weekly.py` 从 313 行降至 282 行，预算同步降为 282；纯文本格式化迁移到 53 行的 `report_content.py`，免交逻辑位于 84 行的 `submission_roster.py`。
- 隐私：差异扫描未发现用户提供的真实免交身份或此前暴露的凭据片段；没有读取 `config/.env`、`output/`、`logs/` 或真实业务数据。
- 剩余风险：姓名规则无法消除重名风险；掩码标识只有在它确实是通讯录完整显示名时才能精确命中，获得稳定 userid 后应迁移为 `USERID:`。
- 延后工作：由管理员按需重跑目标周，重新生成历史统计产物。
