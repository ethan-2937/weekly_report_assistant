# 产品契约

## 目标

工作周报汇总与评价系统把钉钉通讯录和周报转化为可复核的提交统计、员工工作摘要、团队负责人履职检查和管理协调事项，并通过受权限保护的 Web 页面展示。

## 用户

- 完整报告查看者：查看跨部门提交、AI评价、下载和采集任务。
- 范围用户：只查看已授权部门、人员或 userid 的数据。
- 管理员：管理系统账号和权限范围，不因 `ADMIN` 自动获得完整周报内容。
- 员工与负责人：是被分析对象，不等于系统登录用户。

## 业务不变量

1. 默认处理上一完整 ISO 周，业务周期为周一至周日；该周报告的提交归属窗口为本周周四 00:00 至下一周周四 00:00 前，所有时间使用 Asia/Shanghai。
2. 应交人员来自授权通讯录或明确名单，不能从实交周报反推；明确配置的免交人员在比较前从应交与实交统计中同时排除。
3. 人员匹配优先使用稳定 ID，重名和无法匹配进入待确认。
4. 模板合规性只依据真实钉钉模板；原始周报出现新条件字段后，其他岗位按通用 4 项、销售按 9 项、技术/产品按 11 项必填字段计算，岗位适用性来自通讯录职务且不做模糊猜测，历史模板不被新规则倒查降分；管理评价使用独立筛选标准。
5. 负责人个人周报和团队额外职责分别评价；负责人附件下载到目标周私有目录后由 Codex 解析，下载或解析失败必须给出明确状态，不能误判为未提交或留空。
6. 需要协调或拍板的事项在管理报告中置顶，但没有风险字段时不判模板不合格。
7. 原始数据默认只保存在运行目录，不进入 Git 或普通日志。
8. 后端权限决定可见数据，前端只负责呈现。
9. 周一至周三补交的周报归入上一业务周；周四 00:00 起提交的周报归入新的业务周，归属窗口不得重叠。
10. 每周日 18:00 的自动检查面向当前业务周，只是一条提交提醒而不是最终逾期结论；未交人员收到私人工作通知，配置的管理员无论任务成功或失败都收到不含人员明细的执行结果。
11. 每周一 12:00 的个人评价通知面向上一完整业务周，只发送给已提交且私有反馈通过 Harness 校验的人员；消息先肯定证据充分的优点，再重点给出改进建议，随后用两句式暖心感谢收束并附服务器私有配置的 HR 联系方式。感谢语优先依据本人当周具体贡献，第二句使用“团队因您”表达团队价值；旧版反馈安全回退为通用感谢。未提交、免交、评价未就绪或身份不确定时不得发送。
12. 管理员可向服务器配置的单一反馈接收人试发周日提醒和周一评价样例；试发不得读取真实候选或评价，不得写正式幂等状态，也不得提供任意 userid 输入。
13. 管理员可按单个周次复核已经完整发送的周一评价通知正文；该能力只用于通知审计，不授予原始周报或完整管理报告权限。只有发送状态为完整、人数一致且当前受校验反馈能与发送时摘要或旧状态时间一致时才展示正文。

## 输出契约

```text
output/contacts/                         # 授权通讯录缓存
output/<YYYY-Www>/raw/reports.json       # 指定模板原始周报，敏感，用于统计和评价
output/<YYYY-Www>/raw/all_reports.json   # 新旧周报模板完整导出快照，敏感
output/<YYYY-Www>/attachments/team_leads/ # 负责人附件，敏感，仅本地
output/<YYYY-Www>/exports/submission_status.csv
output/<YYYY-Www>/exports/project_details.csv # 项目明细；身份字段仅供服务层权限过滤
output/<YYYY-Www>/exports/leader_subordinates.csv # 负责人下属确认表，敏感，仅本地
output/<YYYY-Www>/analysis/analysis_input.md
output/<YYYY-Www>/summary/submission_check.md
output/<YYYY-Www>/summary/manager_report.md
output/<YYYY-Www>/automation/evaluation_state.json # 自动评价状态，仅含摘要和散列
output/<YYYY-Www>/automation/employee_feedback.json # 私有个人反馈，含稳定 userid、精简评价和暖心感谢
output/<YYYY-Www>/notifications/monday-evaluation-feedback.json # 评价通知幂等状态，仅含阶段、人数、时间和可选反馈摘要
```

`manager_report.md` 是 AI 评价页的正式输入。路径或结构变化属于跨 Python、Skill、Spring 和 Vue 的契约变更。

服务器自动评价从周日 18:10 起采集当前 ISO 周，周一和周二每两小时继续刷新同一目标周，最后一次为周二 22:10；输入无变化时保留有效报告并跳过模型调用。周三 00:00 起不再自动运行，但这不改变周四至下一周周四前的提交归属窗口；之后需要纳入的数据由管理员显式指定周次重跑。状态文件不包含姓名、userid、周报正文或凭据。

## UI 约定

- 登录页使用正式、克制的文案，不出现 HR、驾驶舱或技术证明卡片。
- AI评价页可以使用宽屏，普通页面保持较窄阅读宽度。
- 标准状态标签驱动颜色，颜色不能替代文字结论。
- 提交概览顶部统一展示应交、已交、未交和负责人统计；未交统计是进入授权范围未交名单的交互入口，正文区域分别说明模板合规与管理评价规则。
- 正式报告仍保留五维数据；Web 主视图和员工详情标题使用四维展示语义，`综合结论/需跟进` 继续保留在详情内容中，不得丢弃。
- AI 评价的员工与负责人详情支持收起预览和分区 Excel 导出；导出仅包含当前账号已经获授权的前端报告内容。
- 本周重点从表格或逐人叙述两种员工评价格式提取 AI 红黑榜；人员条目使用已授权提交名单补充部门和职位，不能因模型选择叙述格式而只剩团队事项。
- 项目明细作为独立页面展示序号、姓名及参考模板字段共八列数据；多个客户/项目保留原填写，服务端按当前账号范围过滤后再返回列表或生成 XLSX。
- 提交概览可下载原周报 XLSX；文件按参考格式生成新旧模板两个工作表，保留重复提交，只包含当前账号授权范围。地址、评论、点赞和阅读统计不在接口数据中，不进入导出。W28 可按用户确认直接使用既有指定模板快照，不要求补采旧模板。
- AI 评价中能够与当前周授权人员稳定匹配的姓名可按需打开单人周报原文；同名人员必须按部门/职务确认，未提交或原文不可用时只展示安全状态，其他页面不启用该交互。
- 反馈入口保持简单，账号、页面和环境信息自动附带。
- 运行状态页的通知试发仅对 ADMIN 展示，发送前必须明确确认配置接收人名称。
- 运行状态页的 Feedback 通知复核仅对 ADMIN 展示，按周主动加载并逐人折叠；周次切换时清除旧正文，一致性无法确认时只显示警告。
